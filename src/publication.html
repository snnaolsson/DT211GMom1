Frontend-baserad webbutveckling

Arbetsprocessen

Jag börjar med att installera parcel. Skapar ett repo till projektet. Kopplar repot till netlify (se rubrik publicering).

Jag skapar sedan mina sidor: index.html, publication.html och pictures.html. Jag skapar en CSS fil och js-fil och kopplar dessa till varje html-sida. Jag commitar med jämna mellanrum till min develop-branch och när jag känner att en del av sidan är helt färdig mergar jag develop och main.

Struktur

All HTML, CSS, JS och media som hör till webbsidan ligger i en src-katalog. I den katalogen så finns underkataloger för bilder, css och js. Detta för att ha en lättöverskådlig struktur i arbetet. En gitignore-fil finns till projektet och i den finns de delar som inte ska sparas till git.

Automatisering

Git

Sidan är kopplad till git och har ett github-repo. I repot finns två olika branches, en som heter main och en som heter develop. Under utvecklingsfasen av en ny del eller funktion på webbsidan så commitas alla förändringar till develp branchen. Detta för att inte publicera flera ofärdiga lösnignar till webbplatsen. När funktionen eller delen på sidan är klar och fungerar som den ska så mergas develop med main.

Publicering

Hemsidan publiceras automatiskt via Netlify. Hemsidan är kopplad till mitt repository på github och när "npm run build" körs så uppdateras automatiskt hemsidan utifrån dist-katalogen.
